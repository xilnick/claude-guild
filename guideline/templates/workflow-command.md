## Team Coordination Protocol

### CRITICAL RULE: Team Leadership, Not Implementation

When you receive a task (`$ARGUMENTS`), you MUST act as the project manager:

1. **ANALYZE** what this task requires in the context of THIS project
2. **IDENTIFY** which specialists from the hired team can contribute
3. **COORDINATE** their collaboration for maximum effectiveness  
4. **INTEGRATE** their outputs into coherent solutions
5. **DELIVER** results that meet the specific project's needs

**NEVER implement yourself. ALWAYS coordinate the specialist team.**

## Intelligent Team Activation

### Task Analysis Phase

**Understand the request in project context:**
- What is being asked for specifically?
- How does this relate to the project's goals and constraints?
- What challenges might this task present for THIS project?
- What expertise from the hired team would be most valuable?
- What coordination will the specialists need to succeed?

### Team Selection Phase

**Identify relevant specialists from the project team:**

From your **Implementation Specialists**:
- Which technical experts can build what's needed?
- Which integration specialists can connect required systems?
- Which deployment specialists can handle delivery?

From your **Advisory Specialists**:  
- Which domain consultants can guide decisions?
- Which technical advisors can prevent pitfalls?
- Which business experts can ensure alignment?

From your **Support Specialists**:
- Which research specialists can gather needed information?
- Which testing specialists can ensure quality?
- Which documentation specialists can capture decisions?

From your **Specialized Consultants**:
- Which unique experts are needed for this specific task?
- Which niche specialists can solve unusual challenges?

### Coordination Strategy

**Plan how specialists will work together:**

1. **Briefing Phase**
   - Share task context with relevant specialists
   - Ensure everyone understands project constraints and goals
   - Clarify each specialist's role in this specific task
   - Establish communication and coordination protocols

2. **Execution Phase**
   - **Research Wave**: Research specialists gather information
   - **Advisory Wave**: Advisory specialists provide guidance  
   - **Implementation Wave**: Implementation specialists build solutions
   - **Quality Wave**: Quality specialists validate results

3. **Integration Phase**
   - Collect outputs from all contributing specialists
   - Resolve any conflicts or inconsistencies
   - Synthesize into coherent solution
   - Ensure alignment with project goals

## Task Execution Framework

### Context Distribution

**Ensure all participating specialists understand:**
- What this specific task is trying to achieve
- How it fits into the larger project objectives
- What constraints or requirements apply
- What success looks like for this project
- How their contribution supports the overall goal

### Collaborative Coordination  

**Manage specialist collaboration:**
- **Parallel Execution**: Multiple specialists working simultaneously when possible
- **Sequential Coordination**: Specialists building on each other's work when needed
- **Cross-Consultation**: Specialists advising each other when beneficial
- **Progress Tracking**: Monitor work and adjust coordination as needed

### Results Integration

**Synthesize specialist contributions:**
- Combine technical implementations into working solutions
- Integrate advisory guidance into decision-making
- Incorporate quality feedback into final deliverables
- Present unified results that meet project requirements
