## Intelligent Hiring Process

### Phase 1: Systematic Project Intelligence

**Execute comprehensive project discovery using structured analysis:**

#### 1. **Core Project DNA Analysis**

**Project Classification:**
- **Type**: Determine project category (web app, API service, library, CLI tool, mobile app, data platform, etc.)
- **Scale**: Assess complexity level (prototype, MVP, production system, enterprise platform)
- **Stage**: Identify development phase (greenfield, growth, mature, legacy modernization)
- **Domain**: Classify business sector (fintech, e-commerce, healthcare, education, etc.)

**Architecture Discovery:**
- **Codebase Structure**: Analyze directory structure, module organization, dependency patterns
- **Technical Patterns**: Identify architectural patterns (microservices, monolith, serverless, etc.)
- **Data Flow**: Map how data moves through the system
- **Integration Points**: Discover external service dependencies and APIs

#### 2. **Technology Stack Deep Scan**

**Dynamic Technology Discovery:**
- **Codebase Analysis**: Let intelligence examine all files, configurations, and dependencies
- **Technology Identification**: Discover and categorize whatever technologies are actually present
- **Pattern Recognition**: Identify architectural patterns, design approaches, and technical conventions
- **Dependency Mapping**: Understand all external dependencies and their purposes

**Infrastructure & Operations Discovery:**
- **Data Layer**: Discover all data storage and processing systems in use
- **Service Architecture**: Identify all external services and integrations
- **Deployment Infrastructure**: Understand deployment, orchestration, and operational tools
- **Development Ecosystem**: Map all tools, frameworks, and utilities supporting development

#### 3. **Business Context Intelligence**

**Stakeholder Analysis:**
- **Primary Users**: Who directly interacts with the system and how
- **Business Stakeholders**: Decision makers, product owners, domain experts
- **Technical Team**: Current team structure, expertise levels, knowledge gaps
- **External Dependencies**: Third-party vendors, compliance auditors, integration partners

**Business Requirements:**
- **Core Value Proposition**: Primary business problems solved
- **Success Metrics**: KPIs, performance targets, user satisfaction goals
- **Regulatory Context**: Discovered compliance and regulatory requirements specific to this domain
- **Business Constraints**: Budget, timeline, resource, and discovered regulatory limitations

#### 4. **Challenge & Opportunity Matrix**

**Current Pain Points:**
- **Technical Debt**: Code quality issues, outdated dependencies, architectural problems
- **Performance Issues**: Bottlenecks, scalability limitations, resource constraints
- **Development Velocity**: Process inefficiencies, knowledge silos, tooling gaps
- **Quality Concerns**: Bug rates, testing coverage, reliability issues

**Growth Opportunities:**
- **Quick Wins**: Low-effort, high-impact improvements
- **Strategic Investments**: Major improvements requiring significant effort
- **Innovation Areas**: Emerging technology adoption, competitive advantages
- **Risk Mitigation**: Security vulnerabilities, compliance gaps, disaster recovery

#### 5. **Expertise Gap Analysis**

**Critical Knowledge Needs:**
- **Domain Expertise**: Business logic understanding, user experience optimization
- **Technical Skills**: Missing technical capabilities, emerging technology adoption
- **Process Improvement**: Development workflow optimization, quality assurance
- **Strategic Guidance**: Architecture decisions, technology choices, scaling strategies
