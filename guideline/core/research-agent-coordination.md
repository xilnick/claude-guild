# Research Agent Coordination

Subagent research orchestration for comprehensive project discovery and analysis.

---

## Coordination Philosophy

**Intelligent Agent Orchestration**: Coordinate multiple specialized research agents to perform comprehensive project discovery through parallel investigation, pattern recognition, and adaptive learning.

**Parallel Discovery Strategy**: Maximize discovery efficiency through simultaneous investigation while ensuring comprehensive coverage and accuracy.

---

## Agent Architecture

### Research Agent Types

**1. Discovery Agents**
- **Purpose**: Universal technology investigation and identification
- **Focus**: Surface analysis, technology inference, basic pattern recognition
- **Scope**: All project files and configurations
- **Methods**: File system analysis, content parsing, dependency mapping

**2. Pattern Analysis Agents**
- **Purpose**: Deep analysis of architectural and coding patterns
- **Focus**: Design patterns, conventions, integration approaches
- **Scope**: Code organization, component relationships, architectural decisions
- **Methods**: Pattern recognition, structural analysis, behavioral inference

**3. Integration Mapping Agents**
- **Purpose**: Map system integrations and component connections
- **Focus**: Service relationships, data flow, dependencies
- **Scope**: Component interactions, API integrations, infrastructure connections
- **Methods**: Connection tracing, dependency analysis, flow mapping

**4. Operational Intelligence Agents**
- **Purpose**: Discover deployment and operational patterns
- **Focus**: CI/CD, monitoring, maintenance workflows
- **Scope**: Deployment scripts, configurations, operational procedures
- **Methods**: Operational analysis, workflow mapping, infrastructure discovery

## Coordination Protocols

### Phase-Based Discovery

**Phase 1: Parallel Surface Analysis**
```javascript
Surface Analysis Coordination:
1. **Agent Deployment**
   - Deploy all agent types simultaneously
   - Each agent performs rapid surface scan
   - Initial mapping of project structure
   - Quick identification of major technologies

2. **Parallel Investigation Areas**
   Discovery Agent:
   - Scan all files and directories
   - Identify file types and configurations
   - Map basic technology indicators

   Pattern Analysis Agent:
   - Initial architectural pattern recognition
   - Basic coding convention identification
   - High-level design pattern detection

   Integration Mapping Agent:
   - Map obvious component connections
   - Identify import/dependency relationships
   - Basic service discovery

   Operational Intelligence Agent:
   - Identify deployment files and scripts
   - Recognize CI/CD configurations
   - Map basic operational patterns

3. **Synthesis and Prioritization**
   - Combine initial findings from all agents
   - Identify areas requiring deep investigation
   - Prioritize complex or unknown technologies
   - Plan Phase 2 deep investigation strategy
```

**Phase 2: Specialized Deep Investigation**
```javascript
Deep Investigation Coordination:
1. **Focused Agent Deployment**
   - Deploy agents to priority areas from Phase 1
   - Specialized investigation based on agent expertise
   - Parallel investigation of different project aspects
   - Adaptive investigation based on discoveries

2. **Deep Investigation Areas**
   Discovery Agent (Deep):
   - Investigate unknown technologies and patterns
   - Research unfamiliar configurations and frameworks
   - Analyze complex build systems and dependencies
   - Study proprietary or custom technologies

   Pattern Analysis Agent (Deep):
   - Detailed architectural analysis
   - Complex design pattern identification
   - Advanced coding convention analysis
   - Anti-pattern and improvement opportunities

   Integration Mapping Agent (Deep):
   - Complex integration pattern analysis
   - Service dependency mapping
   - Data flow and state management analysis
   - External service integration mapping

   Operational Intelligence Agent (Deep):
   - Detailed deployment workflow analysis
   - Advanced CI/CD pipeline mapping
   - Comprehensive monitoring and logging analysis
   - Security and compliance pattern discovery

3. **Gap Identification**
   - Identify areas with insufficient information
   - Discover unknown technologies requiring research
   - Recognize complex integrations needing deeper analysis
   - Plan Phase 3 focused investigation
```

**Phase 3: Focused Research and Validation**
```javascript
Focused Research Coordination:
1. **Targeted Investigation**
   - Address gaps from Phase 2
   - Research unknown technologies externally
   - Validate discoveries against actual implementation
   - Resolve conflicts between agent findings

2. **External Research Coordination**
   - Deploy research agents for unknown technologies
   - Investigate best practices and patterns
   - Study documentation and community resources
   - Learn about technology integration approaches

3. **Validation and Verification**
   - Cross-validate findings between agents
   - Test inferred patterns against project behavior
   - Verify integration mappings through dependency analysis
   - Confirm operational intelligence with deployment analysis

4. **Synthesis and Intelligence Generation**
   - Combine all agent findings
   - Generate comprehensive project intelligence
   - Extract reusable patterns and best practices
   - Create adaptive agents with discovered expertise
```
